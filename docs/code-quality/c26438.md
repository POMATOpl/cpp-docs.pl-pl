# <a name="c26438-no_goto"></a>C26438 NO_GOTO

"Unikaj `goto`."

**Podstawowe wytyczne dotyczące języka C++**:\
ES.76: Unikaj goto

Stosowanie `goto` jest powszechnie uważane za niebezpieczną i podatną na błędy praktykę. Jest to dopuszczalne tylko w wygenerowanym kodzie, na przykład w analizatorze generowanym na podstawie gramatyki. Dzięki nowoczesnym funkcjom języka C++ i narzędziom dostarczonym przez `goto` bibliotekę obsługi wytycznych powinno być łatwo uniknąć całkowicie.

## <a name="remarks"></a>Uwagi

- Ta reguła ostrzega `goto`o każdym wystąpieniu , nawet jeśli dzieje się to w kodzie martwym, z wyjątkiem kodu szablonu, który nigdy nie jest używany i w związku z tym ignorowane przez kompilator.
- Ostrzeżenia mogą mnożyć, gdy makro zawiera `goto`. Obecne mechanizmy raportowania wskazują wszystkie wystąpienia, w których takie makro zostanie rozwinięte. Często można go naprawić w jednym miejscu, zmieniając makro lub unikając jego stosowania na rzecz bardziej łagodzących mechanizmów.

## <a name="example"></a>Przykład

"goto clean-up" w makro

```cpp
#define ENSURE(E, L) if (!(E)) goto L;

void poll(connection &c)
{
    ENSURE(c.open(), end);                  // C26438

    while (c.wait())
    {
        connection::header h{};
        connection::signature s{};
        ENSURE(c.read_header(h), end);      // C26438
        ENSURE(c.read_signature(s), end);   // C26438
        // ...
    }

end:
    c.close();
}
```

'goto clean-up' w makro - zastąpiony gsl::finally

```cpp
void poll(connection &c)
{
    auto end = gsl::finally([&c] { c.close(); });

    if (!c.open())
        return;

    while (c.wait())
    {
        connection::header h{};
        connection::signature s{};
        if(!c.read_header(h))
            return;
       if(!c.read_signature(s))
            return;
        // ...
    }
}
```
