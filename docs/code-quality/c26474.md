# <a name="c26474-no_implicit_cast"></a>C26474 NO_IMPLICIT_CAST

"Nie rzutuj między typami wskaźników, gdy konwersja może być niejawna."

**Podstawowe wytyczne dotyczące języka C++**: Typ.1: Unikaj rzutów

W niektórych przypadkach rzutki niejawne między typami wskaźników można bezpiecznie wykonać i nie wymagają użytkownika do zapisu określonego wyrażenia rzutu. Ta reguła znajduje wystąpienia takich niepotrzebnych rzutowania, które można usunąć.

## <a name="remarks"></a>Uwagi

- Identyfikator reguły jest nieco mylący: powinien być interpretowany jako "niejawne rzutowanie nie jest używane tam, gdzie jest dopuszczalne".
  - Reguła ma zastosowanie tylko do wskaźników i sprawdza rzutowania statyczne i ponownie interpretuje rzutowania.
  - Następujące przypadki są dopuszczalne konwersje wskaźnik, który nie powinien używać jawnych wyrażeń rzutowań:
  - konwersji na nullptr_t;
  - przekształcenie w pustkę*;
  - konwersji z typu pochodnego na jego bazę.

## <a name="example"></a>Przykład

niepotrzebna konwersja ukrywa błąd logiczny

```cpp
template<class T>
bool register_buffer(T buffer) {
    auto p = reinterpret_cast<void*>(buffer); // C26474, also 26490 NO_REINTERPRET_CAST
    return buffers_.insert(p).second;
}

void merge_bytes(std::uint8_t *left, std::uint8_t *right)
{
    if (left && register_buffer(*left)) { // Unintended dereference!
        // ...
        if (right && register_buffer(right)) {
            // ...
        }
    }
}
```

niepotrzebna konwersja ukrywa błąd logiczny - przerobiony

```cpp
// ...
template<class T>
bool register_buffer(T *buffer) {
    auto p = buffer;
    return buffers_.insert(p).second;
}
// ...
```
