# <a name="c26432-define_or_delete_special_ops"></a>C26432 DEFINE_OR_DELETE_SPECIAL_OPS

"Jeśli zdefiniujesz lub usuniesz dowolną domyślną operację w typie, zdefiniuj lub usuń je wszystkie."

**Podstawowe wytyczne C++**: [C.21: Jeśli zdefiniujesz lub =usuniesz dowolną domyślną operację, zdefiniuj lub =usuń je wszystkie](https://isocpp.github.io/CppCoreGuidelines/CppCoreGuidelines#c21-if-you-define-or-delete-any-default-operation-define-or-delete-them-all)

Zakłada się, że operacje specjalne, takie jak konstruktory, zmieniają zachowanie typów, tak aby bardziej polegały na mechanizmach języka, aby automatycznie wymuszać określone scenariusze. Przykładem kanonicznym jest zarządzanie zasobami. Jeśli jawnie zdefiniować, domyślnie lub usunąć dowolną z tych operacji specjalnych, sygnalizuje, że chcesz uniknąć specjalnej obsługi typu. Jest niespójne, aby pozostawić inne operacje nieokreślone, to znaczy, niejawnie zdefiniowane jako usunięte przez kompilator.

## <a name="remarks"></a>Uwagi

- Kontrola ta implementuje "regułę pięciu", która traktuje następujące operacje jako specjalne:
  - konstruktorów kopii;
  - konstruktory ruchu;
  - operatory przypisywania kopii;
  - przenoszenie operatorów przypisania;
  - Destruktory.
- Reguła nie sprawdza, czy operacje są zdefiniowane w ten sam sposób. Można mieszać usunięte i domyślne operacje z jawnie zdefiniowanymi operacjami. Jednak należy określić wszystkie z nich, jeśli określisz którykolwiek z nich.
- Poziomy dostępu nie są ważne i mogą być również mieszane.
- Ostrzeżenie oznacza pierwszą niestatyczną definicję funkcji typu, raz na typ.
