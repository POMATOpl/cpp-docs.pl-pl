# <a name="c26436-need_virtual_dtor"></a>C26436 NEED_VIRTUAL_DTOR

"Typ z funkcją wirtualną wymaga publicznego wirtualnego lub chronionego niewirtualnego destruktora."

**Podstawowe wytyczne C++**: C.35: Destruktor klasy podstawowej powinien być publiczny i wirtualny lub chroniony i niewirtualny

Jeśli klasa definiuje funkcję wirtualną staje się polimorficzne, co oznacza, że klasy pochodne można zmienić swoje zachowanie, w tym zarządzania zasobami i logiki niszczenia. Ponieważ kod klienta może wywoływać typy polimorficzne za pośrednictwem wskaźników do klas podstawowych, nie ma możliwości, aby klient jawnie wybrał, które zachowanie jest odpowiednie bez przesunięcia w dół. Aby upewnić się, że zasoby są zarządzane konsekwentnie i zniszczenia występuje zgodnie z regułami rzeczywistego typu zaleca się zdefiniowanie publicznego wirtualnego destruktora. Jeśli hierarchia typów jest przeznaczona do nie zezwalania na kod klienta do bezpośredniego niszczenia obiektów, destruktory powinny być zdefiniowane jako chronione niewirtualne.

## <a name="remarks"></a>Uwagi

- Ostrzeżenie pojawia się w pierwszej definicji funkcji wirtualnej typu (może to być wirtualny destruktor, jeśli nie jest publiczny), raz na typ.
  - Ponieważ definicja może być umieszczona oddzielnie od deklaracji, nie zawsze może mieć którykolwiek z specyfikatorów wirtualnych. Ale ostrzeżenie jest nadal ważne - sprawdza rzeczywistą "wirtualność" funkcji.
