# <a name="c26409-no_new_delete"></a>C26409 NO_NEW_DELETE

Nawet jeśli kod jest czysty w przypadku wywołania funkcji malloc () i Free (), nadal sugerujemy, aby wziąć pod uwagę lepsze opcje niż jawne użycie operatorów [New i DELETE](/cpp/cpp/new-and-delete-operators).

  **Podstawowe wytyczne dotyczące języka C++**: [R. 11: Unikaj wywołania funkcji New i DELETE jawnie](https://isocpp.github.io/CppCoreGuidelines/CppCoreGuidelines#r11-avoid-calling-new-and-delete-explicitly)

Ostatecznym rozwiązaniem jest rozpoczęcie korzystania z inteligentnych wskaźników z odpowiednimi funkcjami fabrycznymi, takimi jak [std:: make_unique](/cpp/standard-library/memory-functions#make_unique).

## <a name="remarks"></a>Uwagi

- Narzędzie sprawdzania ostrzega o wywołaniach dowolnego rodzaju operatora **`new`** lub **`delete`** : skalarnej, wektorowej, przeciążonej wersji (globalnym i specyficznym dla klasy), a także o wersjach umieszczania. Ten ostatni przypadek może wymagać pewnych wyjaśnień dotyczących podstawowych wytycznych dotyczących sugerowanych poprawek i można go pominąć w przyszłości.
