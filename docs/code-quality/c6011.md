# <a name="c6011"></a>C6011

> Ostrzeżenie C6011: wyłuskanie wskaźnika o wartości NULL \<nazwa >

To ostrzeżenie wskazuje, że kod odwołuje się do potencjalnie pustego wskaźnika. Jeśli wartość wskaźnika jest nieprawidłowa, wynik jest niezdefiniowany. Aby rozwiązać ten problem, zweryfikuj wskaźnik przed użyciem.

## <a name="example"></a>Przykład

Poniższy kod generuje to ostrzeżenie, ponieważ wywołanie `malloc` może zwrócić wartość null, jeśli dostępna jest niewystarczająca ilość pamięci:

```cpp
#include <malloc.h>

void f( )
{
  char *p = ( char * ) malloc( 10 );
  *p = '\0';

  // code ...
 free( p );
}
```

Aby poprawić to ostrzeżenie, należy zapoznać się ze wskaźnikiem wartości null, jak pokazano w poniższym kodzie:

```cpp
#include <malloc.h>
void f( )
{
  char *p = ( char * )malloc ( 10 );
  if ( p )
  {
    *p = '\0';
    // code ...

    free( p );
  }
}
```

Funkcje mogą mieć parametry mające adnotację przy użyciu właściwości `Null` w warunku `Pre`. Przydziel pamięć w tych funkcjach przed usunięciem odwołania do parametru. Poniższy kod generuje Ostrzeżenie C6011, ponieważ podjęto próbę cofnięcia odwołania do pustego wskaźnika (`pc`) wewnątrz funkcji bez wcześniejszego przydzielenia pamięci:

```cpp
#include <sal.h>
using namespace vc_attributes;
void f([Pre(Null=Yes)] char* pc)
{
  *pc='\0'; // warning C6011 - pc is null
  // code ...
}
```

Nieostrożnej użycie `malloc` i `free` prowadzi do przecieków pamięci i wyjątków. Aby całkowicie zminimalizować ten rodzaj przecieków i problemów z wyjątkami, unikaj samodzielnego przydzielania pamięci surowej. Zamiast tego należy użyć mechanizmów zapewnianych przez C++ standardową bibliotekę (STL). Należą do nich [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)i [wektor](/cpp/standard-library/vector). Aby uzyskać więcej informacji, zobacz [inteligentne wskaźniki](/cpp/cpp/smart-pointers-modern-cpp) i [ C++ standardowa biblioteka](/cpp/standard-library/cpp-standard-library-reference).
