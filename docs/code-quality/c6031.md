# <a name="c6031"></a>C6031

> Ostrzeżenie C6031: wartość zwracana została zignorowana: *wywołana — funkcja* może zwracać nieoczekiwaną wartość

To ostrzeżenie wskazuje, że obiekt wywołujący nie sprawdza wartości zwracanej funkcji dla błędu. W zależności od tego, która funkcja jest wywoływana, usterka może prowadzić do pozornie przypadkowego niedziałania programu. Obejmuje to awarie i uszkodzenie danych w warunkach błędów lub małych zasobach.

Ogólnie rzecz biorąc, nie jest bezpieczne założenie, że wywołania funkcji wymagające pomyślnego zaistnienia dysku, sieci, pamięci lub innych zasobów. Obiekt wywołujący powinien zawsze sprawdzać wartość zwracaną i odpowiednio obsługiwać przypadki błędów. Należy również rozważyć użycie adnotacji `_Must_inspect_result_`, która sprawdza, czy wartość jest sprawdzana w przydatny sposób.

## <a name="example"></a>Przykład

Poniższy kod generuje to ostrzeżenie:

```cpp
#include <stdio.h>
void f( )
{
    fopen( "test.c", "r" ); // C4996, C6031 return value ignored
    // code ...
}
```

Aby poprawić to ostrzeżenie, należy sprawdzić wartość zwracaną przez funkcję, jak pokazano w poniższym kodzie:

```cpp
#include <stdio.h>
void f( )
{
    FILE *stream;
    if ( (stream = fopen( "test.c", "r" )) == NULL )
        return;
    // code ...
}
```

Poniższy kod używa bezpiecznego działania `fopen_s`, aby poprawić to ostrzeżenie:

```cpp
#include <stdio.h>
void f( )
{
    FILE *stream;
    errno_t err;

    if ( (err = fopen_s( &stream, "test.c", "r" )) !=0 )
    {
        // code ...
    }
}
```

To ostrzeżenie jest również generowane, jeśli obiekt wywołujący ignoruje wartość zwracaną funkcji z adnotacją `_Check_return_`, jak pokazano w poniższym kodzie.

```cpp
#include <sal.h>
_Check_return_ bool func();

void test_f()
{
    func(); //  Warning C6031
}
```

Aby skorygować poprzednie ostrzeżenie, sprawdź wartość zwracaną, jak pokazano w poniższym kodzie:

```cpp
#include <sal.h>
_Check_return_ bool func();

void test_f()
{
    if ( func() ) {
        // code ...
    }
}
```
