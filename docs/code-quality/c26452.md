# <a name="arithmetic-overflow-left-shift-count-is-negative-or-greater-than-or-equal-to-the-operand-size-which-is-undefined-behavior"></a>Przepełnienie arytmetyczne: liczba przesunięcia w lewo jest ujemna lub większa lub równa rozmiarowi operandu, który jest zachowaniem nieokreślonym

To ostrzeżenie wskazuje, że liczba przesunięcia jest ujemna lub większa lub równa liczbie bitów argumentu, które są przesuwane, co powoduje niezdefiniowane zachowanie.
Uwaga: C4293 jest podobne sprawdzenie w kompilatorze Microsoft C++.

## <a name="example"></a>Przykład

```cpp
unsigned __int64 combine(unsigned lo, unsigned hi)
{
  return (hi << 32) | lo; // C26252 here
}
```

Aby poprawić to ostrzeżenie, użyj następującego kodu:

```cpp
unsigned __int64 combine(unsigned lo, unsigned hi)
{
  return ((unsigned __int64)hi << 32) | lo; // OK
}
```
