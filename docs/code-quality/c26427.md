# <a name="c26427-no_global_init_externs"></a>C26427 NO_GLOBAL_INIT_EXTERNS

"Globalny inicjator uzyskuje dostęp do obiektu extern."

**Podstawowe wytyczne dotyczące języka C++**: I.22: Unikaj skomplikowanej inicjalizacji obiektów globalnych

Obiekty globalne mogą być inicjowane w niespójnej lub niezdefiniowanej kolejności, co oznacza, że współzależność między nimi jest ryzykowna i należy jej unikać. Niniejsza wytyczna ma zastosowanie, gdy inicjatory odnoszą `extern`się do innego obiektu, który jest uważany za .

## <a name="remarks"></a>Uwagi

Obiekt jest uważany `extern` za zgodny z następującymi regułami:

- jest to zmienna globalna oznaczona specyfikatorem `extern` lub statycznym członkiem klasy;
- nie znajduje się w anonimowej przestrzeni nazw;
- nie jest oznaczony `const`jako ;
- Elementy członkowskie klasy statycznej są uważane za globalne, więc ich inicjatory są również sprawdzane.

## <a name="example"></a>Przykład

sprawdzanie wersji zewnętrznej

```cpp
// api.cpp
int api_version = API_DEFAULT_VERSION; // Assume it can change at run time, hence non-const.

// client.cpp
extern int api_version;
bool is_legacy_mode = api_version <= API_LEGACY_VERSION; // C26427, also stale value
```

kontrola wersji zewnętrznej – bardziej niezawodna

```cpp
// api.cpp
int api_version = API_DEFAULT_VERSION; // Assume it can change at run time, hence non-const.

// client.cpp
extern int api_version;
bool is_legacy_mode() noexcept
{
    return api_version <= API_LEGACY_VERSION;
}
```
