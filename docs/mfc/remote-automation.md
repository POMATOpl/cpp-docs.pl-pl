# <a name="remote-automation"></a>Automatyzacja zdalna
> [!NOTE]
>  Automatyzacja zdalna to technologia przestarzały i nie powinna być używana dla nowych aplikacji. Visual C++ nie obsługuje system operacyjny Windows 95. Opisano w przypadkach, w których trzeba będzie obsługiwać automatyzacji zdalnej [gdzie jest zdalny automatyzacji mieści się w](where-does-remote-automation-fit-in-q.md).  
  
 Automatyzacja zdalna jest typem [automatyzacji](../mfc/automation.md) umożliwiająca konsumenta interfejsu do wykonania dostawcy interfejsu, który znajduje się na innym komputerze, na przykład w sieci.  
  
 W tym artykule opisano, jak utworzyć obiekty automatyzacji, które mogą być wywoływane i wykonywane zdalnie oraz sposobu tworzenia kontrolerów automatyzacji, które mogą używać tych obiektów automatyzacji zdalnej. Również sprawdza opcji konfiguracji i wskazuje głównych różnic między automatyzacji zdalnej i DCOM (wersja rozproszonego modelu COM i OLE umożliwiający interfejsami spoza dotyczące automatyzacji wywoływane i wykonywane zdalnie).
