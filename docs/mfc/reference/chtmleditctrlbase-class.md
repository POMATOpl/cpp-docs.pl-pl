##  <a name="addtoglyphtable"></a>  CHtmlEditCtrlBase::AddToGlyphTable

Dodaje wpis do tabeli symboli, która określa obrazów do wyświetlania dla określonych tagów w trybie projektowania.

```
HRESULT AddToGlyphTable(
    LPCTSTR szTag,
    LPCTSTR szImgUrl,
    unsigned short nTagType,
    unsigned short nAlignment,
    unsigned short nPosInfo,
    unsigned short nDirection,
    unsigned int nImgWidth,
    unsigned int nImgHeight) const;
```

### <a name="parameters"></a>Parametry

*szTag*<br/>
Nazwa tagu (na przykład, "P" lub "table").

*szImgUrl*<br/>
Adres URL obrazu.

*nTagType*<br/>
Typ znacznika: wartość 0 oznacza, że obraz jest używany tylko w znaczniku otwierającym. 1 oznacza, że obraz jest tylko za pomocą tagu zamykającego. 2 oznacza, że obraz jest otwierającym i zamykającym tagiem. Pojedynczy tagów, takich jak br i komentarz musi zostać dodany z typem tagu ustawione na 0.

*nAlignment*<br/>
Wyrównanie (tylko elementy prostokątne): Ten parametr wskazuje, że obraz jest element z atrybutem wyrównania. Lewy = 0, Centrum = 1, prawy = 2 i undefined = 3. W lewo, prawo lub atrybutów Centrum musi być jawnie ustawione dla elementu.

*nPosInfo*<br/>
Pozycjonowanie informacji. Określa, jakie kaskadowe arkusze stylów (CSS) wartość pozycjonowania glif, których dotyczy przypadku, gdy jest to statyczne pozycjonowanie = 0, pozycjonowanie absolutne = 1, pozycjonowanie względne = 2, a wszystkie = 3. W tym polu można określić jeden symbol tagu, gdy nie jest ustawiony i inny symboli, aby wyświetlać punktu kontrolnego, gdy ustawiony jest tag.

*nDirection*<br/>
Kierunek. Ten parametr określa obraz dla tagu, na podstawie kolejności czytania bieżącego języka. 0 określa od lewej do prawej, 1, określa od prawej do lewej, określa od góry do dołu, 2, 3 określa od dołu do góry i 4 określa wszystkie. To pole jest zazwyczaj ustawiona do 4.

*nImgWidth*<br/>
Szerokość obrazu w pikselach.

*nImgHeight*<br/>
Wysokość obrazu w pikselach.

### <a name="return-value"></a>Wartość zwracana

Zwraca wartość S_OK w przypadku powodzenia lub błędu HRESULT w przypadku niepowodzenia.

### <a name="remarks"></a>Uwagi

Aby uzyskać więcej informacji na temat parametrów, zobacz "Format ciągu w tabeli symbol" w [za pomocą edycji symbole](/previous-versions/windows/internet-explorer/ie-developer/platform-apis/aa969614\(v=vs.85\)).

Ta metoda wysyła [identyfikator polecenia IDM_ADDTOGLYPHTABLE](/previous-versions/aa769891\(v=vs.85\)) do formantu WebBrowser.
