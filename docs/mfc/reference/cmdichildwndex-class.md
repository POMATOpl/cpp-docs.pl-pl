# <a name="cmdichildwndex-class"></a>Klasa CMDIChildWndEx

`CMDIChildWndEx` Klasa oferuje funkcje Windows okna podrzędnego interfejsu (MDI) w usłudze wielu dokumentów. Rozszerza funkcjonalność [klasa CMDIChildWnd](../../mfc/reference/cmdichildwnd-class.md). Struktura wymaga tej klasy, gdy aplikacja MDI używa niektórych klas MFC.

Aby uzyskać więcej informacji, zobacz kod źródłowy znajdujący się w **VC\\atlmfc\\src\\mfc** folder instalacji programu Visual Studio.

## <a name="syntax"></a>Składnia

```
class CMDIChildWndEx : public CMDIChildWnd
```

##  <a name="canshowonwindowslist"></a>  CMDIChildWndEx::CanShowOnWindowsList

Określa, czy nazwa okna podrzędnego MDI mogą być wyświetlane w [klasa CMFCWindowsManagerDialog](../../mfc/reference/cmfcwindowsmanagerdialog-class.md) okno dialogowe.

```
virtual BOOL CanShowOnWindowsList();
```

### <a name="return-value"></a>Wartość zwracana

Wartość TRUE, jeśli okno może być wyświetlany w **Windows** okno dialogowe; w przeciwnym razie wartość FALSE.

### <a name="remarks"></a>Uwagi

Należy przesłonić tę metodę w klasie pochodnej i zwraca wartość FALSE, jeśli okno nie powinien być wyświetlany w **Windows** okno dialogowe. Ta funkcja jest wywoływana z `CMFCWindowsManagerDialog`.

##  <a name="dockpaneleftof"></a>  CMDIChildWndEx::DockPaneLeftOf

Doków jedno okienko po lewej stronie okienka innego.

```
BOOL DockPaneLeftOf(
    CPane* pBar,
    CPane* pLeftOf);
```

### <a name="parameters"></a>Parametry

*pBar*<br/>
Wskaźnik do okienka, który ma być zadokowane.

*pLeftOf*<br/>
Wskaźnik do okienka, która służy jako punkt odniesienia.

### <a name="return-value"></a>Wartość zwracana

Wartość TRUE w przypadku powodzenia, wartość FALSE w przypadku niepowodzenia.

### <a name="remarks"></a>Uwagi

Ta metoda przyjmuje okienka określony przez *pBar* i dokowane go po lewej stronie okienka określony przez *pLeftOf*.

Wywołaj tę metodę, gdy chcesz zadokować kilku okienek, w kolejności wstępnie zdefiniowane.

##  <a name="enableautohidepanes"></a>  CMDIChildWndEx::EnableAutoHidePanes

Umożliwia automatyczne ukrywanie okienka Tryb zadokowany na określonej stronie okna.

```
BOOL EnableAutoHidePanes(DWORD dwDockStyle);
```

### <a name="parameters"></a>Parametry

*dwDockStyle*<br/>
[in] Określa strony ramki głównego okna, który jest włączony. Użyj co najmniej jeden z następujących flag.

- CBRS_ALIGN_LEFT

- CBRS_ALIGN_RIGHT

- CBRS_ALIGN_TOP

Wywoływane przez platformę, kiedy zachodzi potrzeba uzyskania mapy bitowej dla podglądu na żywo elementu podrzędnego MDI.

```
virtual HBITMAP OnGetIconicLivePreviewBitmap(
    BOOL bIsMDIChildActive,
    CPoint& ptLocation);
```

### <a name="parameters"></a>Parametry

*bIsMDIChildActive*<br/>
Ten parametr ma wartość TRUE, jeśli mapa bitowa jest wymagany dla elementu podrzędnego MDI, która jest obecnie aktywna i okna głównego nie jest zminimalizowany. Domyślna, w tym przypadku przetwarzania tworzy migawkę głównego okna.

*ptLocation*<br/>
Określa lokalizację mapy bitowej w głównym (najwyższy poziom) współrzędne klienta okna. Ten punkt powinny być dostarczone przez obiekt wywoływany.

### <a name="return-value"></a>Wartość zwracana

Przetwarzanie, zwraca uchwyt do mapy bitowej 32 bitowej prawidłową, w przeciwnym razie wartość NULL.

### <a name="remarks"></a>Uwagi

Należy przesłonić tę metodę w klasie pochodnej i zwraca prawidłową 32 bitowej mapy bitowej dla podglądu na żywo elementu podrzędnego MDI. Ta metoda jest wywoływana tylko wtedy, gdy elementu podrzędnego MDI jest wyświetlane na kartach zadań Windows 7. Po powrocie NULL MFC wywołuje programy obsługi domyślne i uzyskuje się za pomocą mapy bitowe `PrintClient` lub `PrintWindow`.
