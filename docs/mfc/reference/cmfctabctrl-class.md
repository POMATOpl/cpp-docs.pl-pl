# <a name="cmfctabctrl-class"></a>Klasa CMFCTabCtrl

Klasa `CMFCTabCtrl` zapewnia funkcjonalność formantu karty. Kontrolka tabulacji wyświetla okno dokowane z płaskimi lub trójwymiarowymi kartami u góry lub u dołu. Karty mogą wyświetlać tekst i obraz oraz zmieniać kolor, gdy są aktywne.

## <a name="syntax"></a>Składnia

```
class CMFCTabCtrl : public CMFCBaseTabCtrl
```

## <a name="remarks"></a>Uwagi

Klasa `CMFCTabCtrl` obsługuje:

- Style sterowania tabulatorami, które zawierają 3D, płaskie i płaskie z udostępnionym poziomym paskiem przewijania.

- Karty znajdujące się u góry lub u dołu okna.

- Karty wyświetlane w tekście, obrazach lub tekście i obrazach.

- Karty, które zmieniają kolor, gdy karta jest aktywna.

- Zmiany rozmiaru obramowania dla regulowanych kart.

- Odłączane okna z kartami.

Klasa `CMFCTabCtrl` może być używana z oksem w oknie dialogowym, ale jest przeznaczona dla aplikacji korzystających z pasków sterowania dokowania, takich jak Microsoft Excel i Visual Studio. Aby uzyskać więcej informacji, zobacz [CDockablePane Class](../../mfc/reference/cdockablepane-class.md).

Wykonaj następujące kroki, aby dodać kontrolkę karty o zmiennym rozmiarze, dokowania w aplikacji:

1. Utwórz [wystąpienie klasy CTabbedPane](../../mfc/reference/ctabbedpane-class.md).

1. Wywołanie [CDockablePane::Create](../../mfc/reference/cdockablepane-class.md#create).

1. Użyj [CBaseTabbedPane::AddTab](../../mfc/reference/cbasetabbedpane-class.md#addtab) lub [CMFCBaseTabCtrl::InsertTab,](../../mfc/reference/cmfcbasetabctrl-class.md#inserttab) aby dodać nowe karty.

1. Wywołanie [CBasePane::EnableDocking](../../mfc/reference/cbasepane-class.md#enabledocking) tak, że bieżąca kontrolka karty dokowania może zadokować w oknie ramki głównej.

1. Wywołanie [CFrameWndEx::DockPane](../../mfc/reference/cframewndex-class.md#dockpane) do dokowania okna z kartami w ramce głównej.

Na przykład, jak utworzyć okno z kartami jako pasek sterowania dokowania, zobacz [CTabbedPane Class](../../mfc/reference/ctabbedpane-class.md). Aby `CMFCTabCtrl` użyć jako formantu niedościskowego, utwórz `CMFCTabCtrl` obiekt, a następnie wywołanie [cmfctabctrl::Create](#create).

## <a name="cmfctabctrlcreate"></a><a name="create"></a>CMFCTabCtrl::Tworzenie

```
BOOL Create(

    const RECT& rect,
    CWnd* pParentWnd,
    UINT nID,
    Location location=LOCATION_BOTTOM,
    BOOL bCloseBtn=FALSE);
```

### <a name="parameters"></a>Parametry

* — styl*<br/>
[w] Styl kontrolki karty. Aby uzyskać więcej informacji zobacz uwagi.

*Rect*<br/>
[w] Prostokąt, który ogranicza formant karty.

*pParentWnd*<br/>
[w] Wskaźnik do okna nadrzędnego. Nie może być null.

*Nid*<br/>
[w] Identyfikator kontrolki karty.

*Lokalizacji*<br/>
[w] Lokalizacja kart. Wartość domyślna to LOCATION_BOTTOM. Aby uzyskać więcej informacji zobacz uwagi.

*bCloseBtn*<br/>
[w] PRAWDA, aby wyświetlić przycisk zamknięcia na karcie; w przeciwnym razie FALSE. Wartością domyślną jest FAŁSZ.

PRAWDA, jeśli się powiedzie; w przeciwnym razie FALSE.

```
virtual DROPEFFECT OnDragEnter(
    COleDataObject* pDataObject,
    DWORD dwKeyState,
    CPoint point);
```

### <a name="parameters"></a>Parametry

*pDataObject (1000)*<br/>
[w] Wskazuje obiekt danych zawierający dane, które przeciągają użytkownik.

*dwKeyState (Stan klucza)*<br/>
[w] Zawiera stan kluczy modyfikatora. Ten parametr jest kombinacją bitową (OR) następujących wartości: MK_CONTROL, MK_SHIFT, MK_ALT, MK_LBUTTON, MK_MBUTTON i MK_RBUTTON. Aby uzyskać więcej informacji, zobacz sekcję **Parametry komunikatu** [informacje o wejściu myszy](/windows/win32/inputdev/about-mouse-input).

*Punkt*<br/>
[w] Zawiera bieżącą lokalizację kursora we współrzędnych klienta.

### <a name="return-value"></a>Wartość zwracana

Zawsze DROPEFFECT_NONE, co oznacza, że miejsce docelowe upuszczania nie może zaakceptować danych.

### <a name="remarks"></a>Uwagi

Ta metoda służy do obsługi operacji przeciągania i upuszczania. Zastąd w tej metodzie należy zaimplementować własne zachowanie niestandardowe.

Domyślnie ta metoda `CMFCTabCtrl::OnDragOver`wywołuje tylko , który zawsze zwraca DROPEFFECT_NONE.

[w] Identyfikator zasobu mapy bitowej zawierającego listę obrazów.

[w] Przezroczysty kolor obrazu. Części obrazu, które są tym kolorem, będą przezroczyste. Wartością domyślną jest kolor amarantowy, RGB(255,0,255).
