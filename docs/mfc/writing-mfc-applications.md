# <a name="writing-mfc-applications"></a>Pisanie aplikacji MFC

W tym artykule opisano wstępne kroki, które należy wykonać w celu opracowania aplikacji. Najpierw należy zdecydować, jaki rodzaj aplikacji ma być pisany. Niektóre opcje zostały omówione w obszarze [Opcje projektowania aplikacji](../mfc/application-design-choices.md). Czy aplikacja będzie:

- Uruchamianie w Internecie lub intranecie

- Uruchomiona na kliencie lub na serwerze

- Uruchamianie w przeglądarce lub jako aplikacja autonomiczna

- Korzystanie z technologii COM lub Active

- Pobieranie danych przy użyciu usługi WinInet lub monikerów asynchronicznych

Decyzje określają, które klasy są odpowiednie dla Twojej aplikacji. Odpowiedzi pomagają także określić wybrane opcje podczas uruchamiania Kreatora aplikacji, aby rozpocząć konstruowanie aplikacji.

Po dokonaniu wstępnej decyzji projektowej dotyczącej aplikacji internetowej możesz rozpocząć pracę z kreatorem aplikacji. Użyj Kreatora aplikacji, aby utworzyć szkieletową aplikację i zmodyfikować kod zgodnie z opisem w następujących artykułach:

- W przypadku kontrolki ActiveX zobacz [kontrolki ActiveX w Internecie](../mfc/activex-controls-on-the-internet.md).

Poniższe artykuły zawierają również instrukcje ułatwiające rozpoczęcie zadań programistycznych:

- [Wybór projektu aplikacji](../mfc/application-design-choices.md)

- [Monikery asynchroniczne w Internecie](../mfc/asynchronous-monikers-on-the-internet.md)

- [Podstawy WinInet](../mfc/wininet-basics.md)
