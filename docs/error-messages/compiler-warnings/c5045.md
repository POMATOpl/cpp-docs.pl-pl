# <a name="compiler-warning-c5045"></a>C5045 ostrzeżenia kompilatora

> Kompilator wstawi zaradcze dla luki Spectre dla obciążenia pamięci Jeśli przełącznik/qspectre określonych

## <a name="remarks"></a>Uwagi

Ostrzeżenie C5045 pozwala zobaczyć, jakie wzorców w kodzie, że środki zaradcze krokami zaradczymi dla luki, takich jak LFENCE ma zostać wstawiony po [/qspectre](../../build/reference/qspectre.md) określono opcję kompilatora. Dzięki temu można zidentyfikować pliki kodu, których dotyczy problem z zabezpieczeniami. To ostrzeżenie jest wyłącznie informacyjny: nie jest wstawiany środki zaradcze, dopóki nie zostanie ponownie skompilowany przy użyciu **/qspectre** przełącznika. Funkcje C5045 jest niezależna od **/qspectre** Przełącz, aby można było używać ich obu w tej samej kompilacji.

Ostrzeżenie to jest nowa w programie Visual Studio 2017 wersji 15.7 i jest domyślnie wyłączona. Użyj [/Wall](../../build/reference/compiler-option-warning-level.md) włączyć wszystkie ostrzeżenia, które są domyślnie, wyłączone lub __Wn__*n*__5038__ umożliwiające C5045 jako poziom *n* ostrzeżenie. W środowisku IDE, jest domyślnym poziomem ostrzeżeń **/W3** i to ostrzeżenie może zostać włączona w projekcie **stron właściwości** okna dialogowego. Otwórz **właściwości konfiguracji** > **C/C++** > **wiersza polecenia** i **dodatkowe opcje** Dodaj */w35045*, następnie wybierz **OK**. Aby uzyskać więcej informacji, zobacz [są domyślnie wyłączone ostrzeżenia kompilatora](../../preprocessor/compiler-warnings-that-are-off-by-default.md). Aby uzyskać informacji na temat sposobu wyłączania ostrzeżeń według wersji kompilatora, zobacz [ostrzeżenia kompilatora według wersji kompilatora](compiler-warnings-by-compiler-version.md).

## <a name="example"></a>Przykład

Poniższy przykład generuje ostrzeżenie C5045 podczas kompilowania przez Visual Studio 2017 w wersji 15.7 z **/Wall** lub **/w35045** i **/W3** opcje:

```cpp
// C5045.cpp
// Compile with: cl /EHsc /W3 /w35045 C5045.cpp

int G, G1, G2;

__forceinline
int * bar(int **p, int i)
{
    return p[i];
}

__forceinline
void bar1(int ** p, int i)
{
    if (i < G1) {
        auto x = p[i]; // C5045: mitigation here
        G = *x;
    }
}

__forceinline
void foo(int * p)
{
    G = *p;
}

void baz(int ** p, int i)
{
    if (i < G1) {
        foo(bar(p, i + G2));
    }
    bar1(p, i);
}

int main() { }
```

Dane wyjściowe kompilatora, jeśli to ostrzeżenie jest włączona, wygląda mniej więcej tak:


Komunikaty ostrzegawcze pokazują, że środki zaradcze zostałaby wstawiona w wierszu 16. Wymieniono też, że środki zaradcze są potrzebne, ponieważ obciążenie pamięci kanałów informacyjnych indeksu i w wierszu 15 w wierszu 17. Wstawiał odbywa się we paska i bar1, ale środki zaradcze jest efektywne, gdy są umieszczane w wierszu 16.
