# <a name="compiler-warning-level-1-c4473"></a>Kompilator ostrzeżenie (poziom 1) C4473

> "*funkcji*": przekazano za mało argumentów dla symbole zastępcze w ciągu formatu i ich parametry oczekują *numer* ze zmienną liczbą argumentów, ale *numer* podano Brak argumentu ze zmienną liczbą argumentów *indeksu* jest wymagany w ciągu formatu "*format_string*" ten argument jest używany przez specyfikatora konwersji, ten argument jest używany jako szerokość pola ten argument jest używany jako dokładność ten argument jest używany jako rozmiar bufora

Kompilator wykrył niezgodność między liczbą argumentów wymaganych do spełnienia symbole zastępcze w ciągu formatu i liczby argumentów dostarczone. Prawidłowe użycie printf i scanf rodziny funkcji ze zmienną liczbą argumentów wymaga wpisania dowolną liczbę argumentów określonych przez ciąg formatu. Niektóre symbole zastępcze wymagają dodatkowych argumentów, aby określić szerokość, dokładności lub rozmiar buforu, a także argument dla zawartości. Niezgodność zazwyczaj oznacza, że usterki w kodzie.

Aby uzyskać informacji na temat argumenty skojarzone z symbolami zastępczymi ciąg formatu rodziny funkcji printf, zobacz [składnia specyfikacji formatu: funkcje printf i wprintf](../../c-runtime-library/format-specification-syntax-printf-and-wprintf-functions.md). Zobacz dokumentację, aby uzyskać informacje dotyczące funkcji *funkcja*.

To ostrzeżenie jest nowa w programie Visual Studio 2015.

## <a name="example"></a>Przykład

Te przykłady Pokaż ostrzeżenie C4473 może wynikać z ciągiem formatu na dwa sposoby niezgodność z argumentami, a także pokazują, jak rozwiązać problemy.

```cpp
// c4473p.cpp
// compile with: cl /c /W4 c4473p.cpp
#include <stdio.h>

void print_func(int a, float f)
{
    printf_s("This takes %*f arguments.\n", f); // C4473, missing * width argument
    printf_s("This takes %*f arguments.\n", a, f); // fix: supply width argument
}
```

Pierwszy komunikat o błędzie kompilator wykrywa brak argumentu, ale nie można sprawdzić, czy brak argumentu jest szerokość lub zawartości, dzięki czemu zakłada podany argument jest pierwszym szerokości, a brak argumentu jest drugi , specyfikatora konwersji typu. Należy zbadać ciąg formatu w celu określenia rzeczywistego argumentu brakuje.

```cpp
// c4473s.cpp
// compile with: cl /c /W4 c4473s.cpp
#include <stdio.h>

void scan_func(int a, float f)
{
    char name[20];

    scanf_s("%s", &name);                // C4473, missing buffer size argument
    scanf_s("%s", &name, sizeof(name));  // fix: supply buffer size argument
}
```

W tym przykładzie **scanf_s** wymaga dwóch argumentów dla każdego symbolu zastępczego, jeden adres do zapisywania i chwilę, aby dostarczyć rozmiaru pierwszy. Zapoznaj się z dokumentacją dla każdej funkcji biblioteki objaśnienia dotyczące wymaganych argumentów.
