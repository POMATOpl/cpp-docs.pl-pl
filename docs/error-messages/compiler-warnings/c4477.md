# <a name="compiler-warning-level-1-c4477"></a>Kompilator ostrzeżenie (poziom 1) C4477

> "*— funkcja*": ciąg formatu "*ciąg*"wymaga argumentu typu"*typu*", ale ze zmienną liczbą argumentów *numer* ma typ "*typu*"

Kompilator wykrył jest niezgodny z typem argumentu wymagane do zaspokojenia symbol zastępczy w ciągu formatu i typ podany argument. Prawidłowe użycie printf i scanf rodziny funkcji ze zmienną liczbą argumentów wymaga wpisania argumenty typów określona przez ciąg formatu. Niezgodność zazwyczaj oznacza, że usterki w kodzie.

Aby uzyskać informacji na temat argumenty skojarzone z symbolami zastępczymi ciąg formatu rodziny funkcji printf, zobacz [składnia specyfikacji formatu: funkcje printf i wprintf](../../c-runtime-library/format-specification-syntax-printf-and-wprintf-functions.md). Zobacz dokumentację, aby uzyskać informacje dotyczące funkcji *funkcja*.

To ostrzeżenie jest nowa w programie Visual Studio 2015.

## <a name="example"></a>Przykład

Ten przykład pokazuje dwa ostrzeżenia C4477 dla pierwszej funkcji printf_s, gdy dwa argumenty znajdują się za niewłaściwy typ, a także pokazuje, jak rozwiązać problemy.

```cpp
// C4477p.cpp
// compile with: cl /c /W4 C4477p.cpp
#include <stdio.h>

void print_func(int a, float f)
{
    printf_s("This takes int and double arguments %d and %f.\n", f, a); // C4477, expected int then double
    printf_s("This takes int and double arguments %d and %f.\n", a, f); // fix: types in correct order
}
```

Pierwszy komunikat o błędzie, kompilator wykryje, że **double** zostanie podany argument po **int** oczekuje. A **float** automatycznie zostanie podwyższony do **double** na liście argumentów ze zmienną liczbą argumentów. Drugi komunikat o błędzie wskazuje, że kompilator oczekuje **double** argument, ale **int** został dostarczony. Kompilator nie wiadomo, że zamieniono Kolejność argumentów. Należy zbadać ciągu formatowania, aby określić kolejność rzeczywistego argumentu i podaj poprawne typy argumentów.
