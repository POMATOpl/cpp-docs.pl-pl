# <a name="compiler-warning-c5038"></a>Ostrzeżenie kompilatora C5038

> element członkowski danych "*Członek1*"zostanie zainicjowany po element członkowski danych"*członek2*"element członkowski danych"*elementu członkowskiego*"zostanie zainicjowany po klasie bazowej"*element $base_class* '

Elementy członkowskie klasy są inicjowane w kolejności, w której są deklarowane, nie kolejności, w której są wyświetlane na listach inicjatora. To ostrzeżenie wskazuje, że kolejność inicjowania nie jest taka sama jak kolejność deklaracji elementów członkowskich danych ani klas bazowych. Może to prowadzić do niezdefiniowanego zachowania, jeśli Inicjalizacja jednego członka na liście jest zależna od inicjowania elementu członkowskiego, która jest zadeklarowana w dalszej części.

Ostrzeżenie to jest nowa w programie Visual Studio 2017 w wersji 15.3 i jest domyślnie wyłączona. Użyj [/Wall](../../build/reference/compiler-option-warning-level.md) włączyć wszystkie ostrzeżenia, które są domyślnie, wyłączone lub __Wn__*n*__5038__ umożliwiające C5038 jako poziom *n* ostrzeżenie. Aby uzyskać więcej informacji, zobacz [kompilatora ostrzeżenia, są wyłączone domyślnie](../../preprocessor/compiler-warnings-that-are-off-by-default.md). Aby uzyskać informacji na temat sposobu wyłączania ostrzeżeń według wersji kompilatora, zobacz [ostrzeżenia kompilatora według wersji kompilatora](compiler-warnings-by-compiler-version.md).

## <a name="example"></a>Przykład

W poniższym przykładzie Visual Studio 2017 w wersji 15.3 (z /Wall) zgłasza "ostrzeżenie C5038: element członkowski danych"A::y"zostanie zainicjowany po"A::x"element członkowski danych":

```cpp
struct A
{
    A(int a) : y(a), x(y) {} // C5938 Initialized in reverse, y reused
    int x;
    int y;
};
```

Aby rozwiązać ten problem, Rozmieść na liście inicjatora, aby mieć takiej samej kolejności jak deklaracji. Ostrzeżenie podobne jest wywoływane, gdy jeden lub oba inicjatory się odwoływać do członków klasy podstawowej.
