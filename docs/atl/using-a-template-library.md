# <a name="using-a-template-library"></a>Korzystanie z biblioteki szablonów

Szablon jest nieco podobny do makra. Podobnie jak w przypadku makra wywoływanie szablonu powoduje jego rozwinięcie (z odpowiednimi podstawianiem parametrów) do kodu, który został zapisany. Jednak szablon jest kontynuowany w taki sposób, aby umożliwić tworzenie nowych klas opartych na typach, które są przekazywane jako parametry. Te nowe klasy implementują bezpieczne dla typu sposoby wykonywania operacji wyrażone w kodzie szablonu.

Biblioteki szablonów, takie jak ATL, różnią się od tradycyjnych bibliotek klas C++ w tym, że są zwykle dostarczane tylko jako kod źródłowy (lub jako kod źródłowy o małym, obsługiwanym czasie wykonywania) i nie są z natury hierarchiczne lub niekoniecznie takie same. Zamiast wywodząc się z klasy, aby uzyskać wymaganą funkcjonalność, należy utworzyć wystąpienie klasy na podstawie szablonu.
