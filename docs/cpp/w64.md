# <a name="__w64"></a>__w64

To słowo kluczowe specyficzne dla firmy Microsoft jest przestarzałe. W wersjach programu Visual Studio starszej niż Visual Studio 2013 można oznaczyć zmienne, tak aby podczas kompilowania za pomocą [/Wp64](../build/reference/wp64-detect-64-bit-portability-issues.md) kompilator zgłosi ostrzeżenia, które byłyby raportowane w przypadku kompilowania z kompilatorem 64-bitowym.

## <a name="syntax"></a>Składnia

> *wpisz* **__w64** *Identyfikator* __w64

### <a name="parameters"></a>Parametry

*type*<br/>
Jeden z trzech typów, które mogą spowodować problemy w kodzie, który jest przypisywany z 32-bitowego do 64-bitowego kompilatora: **int**, **Long**lub wskaźnika.

*identyfikatora*<br/>
Identyfikator tworzonej zmiennej.

## <a name="remarks"></a>Uwagi

> [!IMPORTANT]
>  Opcja kompilatora [/Wp64](../build/reference/wp64-detect-64-bit-portability-issues.md) i słowo kluczowe **__w64** są przestarzałe w programie Visual Studio 2010 i Visual Studio 2013 i usunięte w Visual Studio 2013. Jeśli używasz opcji kompilatora `/Wp64` w wierszu polecenia, kompilator wystawia ostrzeżenie wiersza polecenia D9002. Słowo kluczowe **__w64** zostało zignorowane w trybie dyskretnym. Zamiast używać tej opcji i słowa kluczowego do wykrywania 64-bitowych problemów z przenośnością, użyj C++ kompilatora firmy Microsoft, który jest przeznaczony dla platformy 64-bitowej. Aby uzyskać więcej informacji, zobacz [Konfigurowanie C++ wizualizacji dla 64-bitowych, elementów docelowych dla procesorów x64](../build/configuring-programs-for-64-bit-visual-cpp.md).

Każdy element typedef, który ma **__w64** na nim musi być 32 bity na procesorach x86 i 64 w wersji x64.

Aby wykrywać problemy z przenośnością przy użyciu wersji programu C++ Microsoft Compiler starszej niż Visual Studio 2010, należy określić słowo kluczowe **__w64** w każdym z typów typedef, które zmieniają rozmiar między 32 bitowym i 64 platformą bitową. Dla dowolnego typu, **__w64** musi występować tylko w 32-bitowej definicji elementu typedef.

W celu zapewnienia zgodności z poprzednimi wersjami **_w64** jest synonimem dla **__w64** , chyba że opcja kompilatora [/za \(Wyłącz rozszerzenia językowe)](../build/reference/za-ze-disable-language-extensions.md) .

Słowo kluczowe **__w64** jest ignorowane, jeśli kompilacja nie korzysta z `/Wp64`.

Więcej informacji o portach do 64-bitowych można znaleźć w następujących tematach:

- [Opcje kompilatora MSVC](../build/reference/compiler-options.md)

- [Przenoszenie kodu 32-bitowego do 64-bitowego kodu](../build/common-visual-cpp-64-bit-migration-issues.md)

- [Konfigurowanie Visual C++ dla wersji 64-bitowych, platformy docelowe x64](../build/configuring-programs-for-64-bit-visual-cpp.md)

## <a name="example"></a>Przykład

```cpp
// __w64.cpp
// compile with: /W3 /Wp64
typedef int Int_32;
#ifdef _WIN64
typedef __int64 Int_Native;
#else
typedef int __w64 Int_Native;
#endif

int main() {
   Int_32 i0 = 5;
   Int_Native i1 = 10;
   i0 = i1;   // C4244 64-bit int assigned to 32-bit int

   // char __w64 c;  error, cannot use __w64 on char
}
```
