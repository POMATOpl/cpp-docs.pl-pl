# <a name="__w64"></a>__w64

To słowo kluczowe specyficzne dla firmy Microsoft jest przestarzałe. W wersjach programu Visual Studio wcześniejszych niż Visual Studio 2013 umożliwia oznaczenie zmiennych, dzięki czemu podczas kompilowania z [/Wp64](../build/reference/wp64-detect-64-bit-portability-issues.md) kompilator zgłosi wszelkie ostrzeżenia, które byłyby zgłaszane podczas kompilowania za pomocą kompilatora 64-bitowego.

## <a name="syntax"></a>Składnia

> *wpisz* **__w64** *identyfikator*

### <a name="parameters"></a>Parametry

*Typu*<br/>
Jeden z trzech typów, które mogą powodować problemy w kodzie są przenoszone z 32-bitowego do kompilatora 64-bitowego: **int**, **long**lub wskaźnik.

*Identyfikator*<br/>
Identyfikator kretowej zmiennej.

## <a name="remarks"></a>Uwagi

> [!IMPORTANT]
> Opcja kompilatora [/Wp64](../build/reference/wp64-detect-64-bit-portability-issues.md) i **__w64** słowo kluczowe są przestarzałe w programie Visual Studio 2010 i visual studio 2013 i usuwane począwszy od programu Visual Studio 2013. Jeśli używasz `/Wp64` opcji kompilatora w wierszu polecenia, kompilator wydaje ostrzeżenie wiersza polecenia D9002. Słowo kluczowe **__w64** jest po cichu ignorowane. Zamiast używać tej opcji i słowa kluczowego do wykrywania problemów z przenoszeniem 64-bitowych, należy użyć kompilatora Microsoft C++, który jest przeznaczony dla platformy 64-bitowej. Aby uzyskać więcej informacji, zobacz [Konfigurowanie visual C++ dla 64-bitowych obiektów docelowych x64](../build/configuring-programs-for-64-bit-visual-cpp.md).

Każdy typedef, który **ma __w64** na nim musi być 32 bity na x86 i 64 bitów na x64.

Aby wykryć problemy z przenośnością przy użyciu wersji kompilatora Microsoft C++ wcześniej niż Visual Studio 2010, **__w64** słowo kluczowe należy określić na wszystkie typedefs, które zmieniają rozmiar między 32 bit i 64-bitowych platform. Dla każdego takiego typu **__w64** musi pojawić się tylko w 32-bitowej definicji typedef.

W przypadku zgodności z poprzednimi wersjami **_w64** jest synonimem **__w64** chyba że określono opcję kompilatora [/Za \(Wyłącz rozszerzenia języka).](../build/reference/za-ze-disable-language-extensions.md)

**__w64** słowo kluczowe jest ignorowane, jeśli kompilacja nie używa `/Wp64`.

Aby uzyskać więcej informacji na temat przenoszenia do 64-bitowego, zobacz następujące tematy:

- [Opcje kompilatora MSVC](../build/reference/compiler-options.md)

- [Przenoszenie kodu 32-bitowego do kodu 64-bitowego](../build/common-visual-cpp-64-bit-migration-issues.md)

- [Konfigurowanie Visual C++ dla wersji 64-bitowych, platformy docelowe x64](../build/configuring-programs-for-64-bit-visual-cpp.md)

## <a name="example"></a>Przykład

```cpp
// __w64.cpp
// compile with: /W3 /Wp64
typedef int Int_32;
#ifdef _WIN64
typedef __int64 Int_Native;
#else
typedef int __w64 Int_Native;
#endif

int main() {
   Int_32 i0 = 5;
   Int_Native i1 = 10;
   i0 = i1;   // C4244 64-bit int assigned to 32-bit int

   // char __w64 c;  error, cannot use __w64 on char
}
```
