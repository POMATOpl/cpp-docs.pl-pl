# <a name="w64"></a>__w64

**Microsoft Specific** 

To słowo kluczowe jest przestarzała. W wersjach programu Visual Studio starszych niż program Visual Studio 2013 w ten sposób można oznaczyć zmiennych, dzięki czemu podczas kompilacji z [/Wp64](../build/reference/wp64-detect-64-bit-portability-issues.md) kompilator będzie zgłaszać wszelkie ostrzeżenia, które będą zgłaszane w przypadku zostały kompilacja z kompilatorem 64-bitowych.  
  
## <a name="syntax"></a>Składnia  
  
> *Typ* **__w64** *identyfikator*  
 
  
### <a name="parameters"></a>Parametry  

*Typ*  
 Jeden z trzech typów, które mogą powodować problemy w kodzie są przenoszone z 32-bitowej do kompilator 64-bitowy: `int`, `long`, lub wskaźnika.  
  
*Identyfikator*  
Na identyfikator zmiennej, które tworzysz.  
  
## <a name="remarks"></a>Uwagi  
  
> [!IMPORTANT]
>  [/Wp64](../build/reference/wp64-detect-64-bit-portability-issues.md) — opcja kompilatora i `__w64` — słowo kluczowe nie są używane w programie Visual Studio 2010 i Visual Studio 2013 i usuwane, począwszy od programu Visual Studio 2013. Jeśli używasz `/Wp64` — opcja kompilatora w wierszu polecenia, problemy kompilatora [D9002 ostrzeżenie wiersza polecenia](http://msdn.microsoft.com/en-us/c58b405b-0f26-434e-b57f-4f05e1ca81e6). `__w64` — Słowo kluczowe dyskretnie jest ignorowana. Zamiast za pomocą tej opcji i słowo kluczowe wykrywać problemy związane z przenośnością 64-bitowych, należy użyć kompilatora Visual C++, przeznaczonego dla platformy 64-bitowej. Aby uzyskać więcej informacji, zobacz [skonfigurować Visual C++ dla 64-bitowy, x64 cele](../build/configuring-programs-for-64-bit-visual-cpp.md).  
  
 Wszelkie element typedef, który ma `__w64` na musi być 32-bitowy na x86 i 64-bitowy na x64.  
  
 Aby wykryć problemy związane z przenośnością przy użyciu wersji kompilatora Visual C++ w starszych niż program Visual Studio 2010, `__w64` — słowo kluczowe powinien być określony na wszystkie definicje typów, który zmienia rozmiar między 32-bitowe i 64-bitowych platform. Dla każdego typu `__w64` musi występować tylko w 32-bitowych definicji typedef.  
  
 `__w64` — Słowo kluczowe jest ignorowana, jeśli kompilacja nie używa `/Wp64`.

## <a name="example"></a>Przykład  
  
```cpp  
// __w64.cpp  
// compile with: /W3 /Wp64  
typedef int Int_32;  
#ifdef _WIN64  
typedef __int64 Int_Native;  
#else  
typedef int __w64 Int_Native;  
#endif  
  
int main() {  
   Int_32 i0 = 5;  
   Int_Native i1 = 10;  
   i0 = i1;   // C4244 64-bit int assigned to 32-bit int  
  
   // char __w64 c;  error, cannot use __w64 on char  
}  
```
