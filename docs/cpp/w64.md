# <a name="w64"></a>__w64

**Microsoft Specific**

This — słowo kluczowe jest przestarzała. W wersjach programu Visual Studio starszych niż program Visual Studio 2013, dzięki temu można oznaczyć zmienne, tak, aby podczas kompilacji z [/Wp64](../build/reference/wp64-detect-64-bit-portability-issues.md) kompilator zgłosi wszelkie ostrzeżenia, które będą zgłaszane w przypadku zostały kompilowanie za pomocą kompilatora 64-bitowych.

## <a name="syntax"></a>Składnia

> *Typ* **__w64** *identyfikator*

### <a name="parameters"></a>Parametry

*type*<br/>
Jednym z trzech typów, które mogą powodować problemy w kodzie są przenoszone z 32-bitowy na 64-bitowego kompilatora: **int**, **długie**, lub wskaźnika.

*Identyfikator*<br/>
Na identyfikator zmiennej, którą tworzysz.

## <a name="remarks"></a>Uwagi

> [!IMPORTANT]
>  [/Wp64](../build/reference/wp64-detect-64-bit-portability-issues.md) — opcja kompilatora i **__w64** — słowo kluczowe są przestarzałe w programie Visual Studio 2010 i Visual Studio 2013, a usuwane począwszy od programu Visual Studio 2013. Jeśli używasz `/Wp64` — opcja kompilatora w wierszu polecenia, kompilator generuje D9002 ostrzeżenie wiersza polecenia. **__W64** — słowo kluczowe jest dyskretnie ignorowana. Zamiast korzystać z tej opcji i słów kluczowych, wykrywać problemy z przenośnością 64-bitowych, należy użyć programu Microsoft C++ kompilatora, który jest przeznaczony dla platformy 64-bitowej. Aby uzyskać więcej informacji, zobacz [Konfigurowanie Visual C++ x64 64-bitowy, obiekty docelowe](../build/configuring-programs-for-64-bit-visual-cpp.md).

Wszelkie typedef, który ma **__w64** na musi być 32-bitowy na x86 i 64-bitowy na x64.

Aby wykryć problemy za pomocą wersji Microsoft C++ starszych niż program Visual Studio 2010, kompilator **__w64** — słowo kluczowe powinny być określone w dowolnej definicje typów, który zmiana rozmiaru między platformami 64-bitowe i 32-bitowe. Dla dowolnego typu **__w64** musi znajdować się tylko w definicji 32-bitowych dla definicji typu.

W celu zgodności z poprzednimi wersjami **_w64** jest synonimem dla **__w64** chyba że — opcja kompilatora [/Za \(Wyłącz rozszerzenia językowe)](../build/reference/za-ze-disable-language-extensions.md) jest określony.

**__W64** — słowo kluczowe jest ignorowana, jeśli kompilacja nie używa `/Wp64`.

Aby uzyskać więcej informacji na temat przenoszenia do 64-bitowej zobacz następujące tematy:

- [Opcje kompilatora MSVC](../build/reference/compiler-options.md)

- [Eksportowanie kodu 32-bitowego do kodu 64-bitowego](../build/common-visual-cpp-64-bit-migration-issues.md)

- [Konfigurowanie Visual C++ dla wersji 64-bitowych, platformy docelowe x64](../build/configuring-programs-for-64-bit-visual-cpp.md)

## <a name="example"></a>Przykład

```cpp
// __w64.cpp
// compile with: /W3 /Wp64
typedef int Int_32;
#ifdef _WIN64
typedef __int64 Int_Native;
#else
typedef int __w64 Int_Native;
#endif

int main() {
   Int_32 i0 = 5;
   Int_Native i1 = 10;
   i0 = i1;   // C4244 64-bit int assigned to 32-bit int

   // char __w64 c;  error, cannot use __w64 on char
}
```
