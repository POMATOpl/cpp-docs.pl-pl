## <a name="microsoft-specific"></a>Specyficzne dla firmy Microsoft

W Microsoft C++ identyfikatory z dwoma podkreśleniami wiodącymi są zarezerwowane dla implementacji kompilatora. W związku z tym Konwencja Microsoft jest poprzedzać słowa kluczowe specyficzne dla Microsoft podwójnym podkreślnikiem. Tych słów nie można używać jako nazw identyfikatorów.

Rozszerzenia Microsoft są domyślnie włączone. Aby upewnić się, że Twoje programy są w pełni zgodne, można wyłączyć rozszerzenia Microsoft określając zgodność ANSI [/Za](../build/reference/za-ze-disable-language-extensions.md) opcji wiersza polecenia (Kompiluj dla zgodności ANSI) podczas kompilacji. Gdy to zrobisz, wyłączane są słowa kluczowe specyficzne dla firmy Microsoft.

Po włączeniu rozszerzeń Microsoft, można użyć słowa kluczowe specyficzne dla firmy Microsoft w swoich programach. Dla zachowania zgodności z ANSI te słowa kluczowe są poprzedzone podwójnym podkreśleniem. Dla zgodności z poprzednimi wersjami, wersje pojedynczym podkreśleniem wszystkich podwójnym podkreśleniem słów kluczowych z wyjątkiem **__except**, **__finally**, **__leave**, i **_ _try** są obsługiwane. Ponadto **__cdecl** jest dostępna z bez wiodącego podkreślenia.

**__Asm** — słowo kluczowe zastępuje C++ `asm` składni. `asm` jest zarezerwowana dla zachowania zgodności z innymi implementacjami C++, ale nie zaimplementowana. Użyj **__asm**.

**__Based** — słowo kluczowe ma ograniczone zastosowanie do 32-bitowych i 64-bitowych docelowej kompilacji.

**END specyficzny dla Microsoft**
