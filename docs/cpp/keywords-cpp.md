## <a name="microsoft-specific"></a>Specyficzne dla firmy Microsoft  
 W Microsoft C++ identyfikatorów o dwóch wiodące znaki podkreślenia są zarezerwowane do implementacji kompilatora. W związku z tym Konwencji firmy Microsoft jest poprzedzać słowa kluczowe specyficzne dla firmy Microsoft z podwójne znaki podkreślenia. Te słowa nie można użyć jako nazwy identyfikatora.  
  
 Rozszerzenia Microsoft są domyślnie włączone. Aby zapewnić pełni dostosowane programy, można wyłączyć rozszerzenia Microsoft określając ANSI zgodnego [/Za](../build/reference/za-ze-disable-language-extensions.md) opcji wiersza polecenia (Kompiluj zgodność ANSI) podczas kompilacji. Gdy to zrobisz, słowa kluczowe specyficzne dla firmy Microsoft są wyłączone.  
  
 Po włączeniu rozszerzenia Microsoft, można użyć słowa kluczowe specyficzne dla firmy Microsoft w programach. Zgodność ANSI słowa kluczowe są poprzedzone podwójnego podkreślenia. Zgodności z poprzednimi wersjami podkreślenia jednym wersje wszystkich podkreślenia o podwójnej precyzji słów kluczowych z wyjątkiem `__except`, `__finally`, `__leave`, i `__try` są obsługiwane. Ponadto `__cdecl` jest dostępne nie początkowe podkreślenia.  
  
 `__asm` — Słowo kluczowe zastępuje C++ `asm` składni. `asm`jest zarezerwowany do zachowania zgodności z innych implementacji C++, ale nie jest zaimplementowana. Użyj `__asm`.  
  
 `__based` — Słowo kluczowe ma ograniczoną zastosowania kompilacje docelowym 32-bitowe i 64-bitowych.
